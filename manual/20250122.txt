Chapter02의 tagis.html 파일을 실행시킨 뒤에, 페이지 소스 보기를 통해 확인했을 때,
우리가 VS Code에서 작성한 내용+a가 여러가지 <태그>들로 덮여있음을 확인할 수 있다.

태그 사용 방식 : 
    <태그이름> 내용 </태그이름> -> 여는 태그와 닫는 태그의 짝(pair)로 이루어져 있다.

내용은 여러 줄에 걸쳐서 나올 수가 있는데, 해당 경우에는
<태그이름>
    내용1
    내용2
    내용3
</태그이름>
의 형태로 작성할 수 있다. -> 한줄 혹은 여러줄에 작성하는 방식을 문법(Syntax)이라고 표현한다.

cf1. <br> 태그와 같이 닫는 태그가 없는 형태도 있다.

cf2. 태그 뒤에 내용을 써주는 경우도 있다. -> 추후 계속 사용될 방식

제목 태그 / 문단 태그

    1. 제목 태그 -> titletag.html
    2. 문단 태그 -> <p></p> paragraph의 약어.
    <p> 시작점 위와 </p> 밑에 공백이 생긴다.
    3. 개행 태그 -> <br>
    
가로줄 태그 / 공백 태그
    1. 가로줄 태그
        : <hr> -> 닫는 태그 없음.
    2. 공백 태그
        : &nbsp; -> 공백을 강제 삽입하는 태그.

텍스트 관련 태그
    1. b 태그 : bold라는 의미로 태그로 감싸진 부분을 볼드체로 표기한다.
    2. strong 태그 : 브라우저 상에서의 효과는 b태그와 동일하나,
        차이점으로, 컴퓨터가 소스 코드를 해석할 때, 중요하다고 인지하도록 하는 태그로서,
        구글 및 검색 사이트에서 <strong> 태그를 보고 검색어 분류를 할 때 이용된다.
    3. i 태그 : italic이라는 의미로 태그로 감싸진 부분을 이탤릭체로 표기한다.
    4. em 태그 : 브라우저 상에서의 효과는 i 태그와 동일하나,
        차이점으로, 컴퓨터가 소스 코드를 해석할 때, 중요하다고 인지하도록 하는 태그로서,
        구글 및 검색 사이트에서 <strong> 태그를 보고 검색어 분류를 할 때 이용된다.
    (실제로는 strong태그와 em태그를 쓰고, 꾸미는 부분은 CSS에서 담당한다.)
    5. 첨자 태그 : sup vs sub 태그 -> 각각 위 첨자, 아래 첨자 (supsubtag.html 참조)
    6. 밑줄 태그 / 취소선 태그
        밑줄 태그 - u 태그 : underline의 줄임말로 밑줄 표시
        취소선 태그 - s 태그 : strike의 줄임말로 취소선 표시 (ustag.html 참조)
        다만 예전에는 u 태그가 밑줄 자체 표시로 html상에서 많이 쓰였으나,
        꾸미기를 CSS가 담당하게 되면서, 특정 테스트가 맞춤법이 틀렸다는 의미로,
        철자 오류 표시 용도 등으로 변하는 중이다.
