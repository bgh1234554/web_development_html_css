3. 다른 곳으로 링크하기 (SpringBoot에서 보러가기 버튼)
    <a> 태그로 링크 이동하기
    형식 :
        <a href="(연결할 주소): target="(링크를 열 곳 옵션)">제목</a>

        속성 정리
            href : hypertext reference의 줄임말. 실제로 우리가 접속할 페이지의 주소
            target: 링크를 어떤 형태로 열 것인지에 관한 옵션
                _self : 보고 있는 창에서 넘어감
                _blank : 새 탭 / 새 창 -> 최근에는 이것이 많이 쓰인다.

---------------------------- 어제 여기까지 ---------------------------------------

    id 값으로 원하는 곳으로 이동하기. (n7idlink.html 참조)
        p 태그들 100개가 있는 중에, 원래대로라면 p들끼리의 기분을 할 수 없으나, id라는 속성을 지정해주고,
        a태그를 사용하여, id들 중 하나로, 그리고 #기호를 통해서 해당 id로 이동을 하였다.\
        
        페이지에 있는 특정한 요소로 이동하고 싶을 때, id값을 사용한 링크를 넣어주게 되면 스크롤이 긴 페이지에서도 원하는 정보가
        있는 곳으로 빠르게 이동이 가능하다.

    태그로 연락처 연결하기 (n8contactstag.html 참조)
        연결하는 것이라, a태그와 연관이 있다.
        형식 :
            <a href="mailto:메일주소">이메일주소</a>
            <a href="tel:전화번호">전화번호</a>

            이상에서 중요한 부분은 속성 href 내에 속성값이 mailto: / tel: 과 같은 지정값이 있다는 점이다.

Chapter04 시작

    <form> 태그 사용
        : 웹 페이지를 사용하다 보면, 아이디와 패스워드를 입력한다거나,
        체크하고 고르는 등의 사용자로부터 입력받아야 하는 정보들이 있다.
        이때 사용하는 것이 <form> 태그이다.

        <form>태그의 속성
            예를 들어 이름과 나이를 입력하는 양식이 있다면, 우리는 키보드로 해당정보를 입력한다.
            이름 폼에는 -> 텍스트를 입력,
            나이 폼에는 -> 숫자를 입력 / 마우스 휠을 돌려 나이를 선택 / 화살표로 입력

        사용자로부터 입력을 받기 위한 양식을 작성하는 태그 전체를 <form>이라고 하고,
        그 속성으로 이름, 나이 입력요소, 제출, 초기화 등 정보를 제출하기 위한 태그들을 포함한다.

        n1formtag.html 참조

        form 내부의 속성
            1. action="" : 우리가 이름 / 나이를 입력했을 때 그 정보를 가져가서 보여주는 결과 페이지
            2. method="" : 가서 보여주도록 method를 고정. POST, GET, PUT, DELETE에 해당하는 HTTP 메서드를 의미.
            3. autocomplete="" : 다시 입력할 때 자동완성을 켤지 끌지 여부를 결정.
        
        <input> 태그 : 일단 종류가 너무 다양해서 n1에서 작성한 부분만 다룰 예정
            1. name="" : HTML 상으로 특별한 의미를 갖는 것은 아니고, 서버에서 각 항목을 받을 때 무엇에 대한 값인지 구분할 수 있도록 정보를 보내는 용도.
                (사람이 알아볼 수 있게 만든 것. 컴퓨터는 id 쓰면 되니까)

        <label> 태그 : 이상의 <input> 태그와 유기적으로 연결되어 있다. <input> 태그에 부여한 id 속성 값을 <label> 태그의 for 속성에서 불러오기 때문에,
        input-name -> label-for

        <button> 태그 : emmet을 이용하여 목록을 확인했을 때, 제출인지 초기화인지 확인이 가능하다.

    <form> 태그 안 요소 그룹화하기
        반장, 부반장, 서기의 이름과 나이를 각각 입력하는 폼.
        커다란 하나의 폼 안에 이름과 나이를 입력하는 폼이 있고, 반장, 부반장, 서기라는 성격에 따라서 소그룹으로 묶을 예정
        비슷한 정보를 그룹별로 묶는 <fieldset> 태그를 사용할 예정.
        (n2fieldset.html 참조)
        fieldset의 속성 disabled : 비활성화
    
    <input> 태그 관련 사용하기
        (n3textinputtag.html 참조)

        type=""
            text: 문자가 출력
            password: 문자가 가려진다
            placeholder: 사용자가 안내 메시지 등을 미리 확인하고 입력할 수 있도록 사전 표기되는 영역
            maxlength: 입력하는 최대 글자 수
            minlength: 입력하는 최소 글자 수 -> 충족하지 않았을 시 경고문이 나온다.
            serach: 텍스트를 입력하면 input창 맨 끝에 x표가 뜨고 클릭하면 초기화된다.
            tel : 모바일에서 사용하면 숫자패드가 default로 뜬다.

        숫자관련 <input>태그 (n4numberinput.html 참조)
            number : 숫자도 입력 가능하고 화살표로 위 아래 이동으로 숫자를 왔다갔다 가능하다.
            range: 슬라이드 바를 왼쪽 오른쪽으로 조절
                step 속성에 숫자 단위를 입력해서 사용할 수 있다.
                -> 움직이는 단위나 숫자를 파악할 수 없어서 자바스크립트와 함께 최소/최댓값 현재값을 비교하는 방식으로 기능을 추가할 예정이다.
                (단독으로 사용하기에는 불편하다는 뜻)
            date: 달력 모양 아이콘을 클릭하면 달력이 나오고, 그 안에서 날짜를 이동하고 선택이 가능하다.
            min/max: 최소, 최댓값. type마다 다르게 사용된다.

        체크 관련 <input>태그 (n5checkinput.html 참조)
            type:
                checkbox: 체크 박스 형태로 사용자 입력을 받는다.
                    checked: 처음 페이지에 들어왔을 때 미리 체크가 된 상태로 보여준다.
                radio: 라디오 버튼 형태로 사용자 입력을 받는다. 같은 name 속성을 가진 radio끼리는 하나만 선택이 가능하다.
                    (참조 파일에서 fruit와 vegetable name속성 참고.)
                value: 결과를 서버로 넘기는 실제 데이터 값. -> 화면에서는 사과/당근 등으로 체크할 수 있지만 프로그래밍 언어상으로
                    적절하지 않을 수 있기 때문에, 컴퓨터도 알아보고 다수의 개발자가 알아볼 수 있는 영문/코드 형식으로 보내줘야 오류가 없다.

        기타 <input> (n6etcinput.html 참조, html 윗부분에 lang 부분으로 크롬이 페이지의 언어를 인식한다.)
            사용자가 어떤 파일을 첨부해서 업로드하거나 혹은 내용을 숨기는 것. 이메일 양식을 작성하여 제출하는 것 등등...

            type:
                file: 파일을 첨부할 때 사용할 수 있다.
                    accept : 원하는 유형의 파일만 업로드할 수 있도록 제어한다.
                    multiple : 파일 여러개를 한번에 올릴 수 있도록 하는 설정.
                hidden: 무언가 숨겨져 있는 상태.
                    사용자로부터 받은 개인정보나 사용자가 알아서는 안되는 등급의 정보 때문.
                    주로 JavaScript와 함쳐서 작성하는 편이다.
                email: 이메일 형식으로 작성하지 않으면 경고창이 뜬다.
                    다만, 아무텍스트에나 @와.을 넣으면 통과되기 때문에 신뢰가 있는 방법은 아니다.
        
        <input> 태그의 공통 특성 (n7commoninput.html 참조)
            value : placeholder와 달리 어떤 default값을 미리 지정함.
            autofocus : 어떤 페이지가 새로고침 되어 있을 때, 클릭하지 않아도 자동으로 어떤 요소가 선택된 상태를 유지하도록 하는 속성.
            read only : 읽기만 가능하고 입력값은 결과로 전송됨.
            disabled : 사용자가 값을 입력할 수 없고 전송조차 되지 않는다.
            requried : 반드시 input 창에 값을 입력해야 하는 것을 뜻한다.
