태그로 여러 요소 나열하기

    목록을 표현하는 태그 <ul> vs <ol>
        1. <ul> : unordered list의 줄임말
            순서가 상관없는 리스트를 만들 때 사용한다.
        2. <ol> : ordered list의 줄임말
            순서가 관계있는 리스트를 만들 때 사용한다.
        3. 1, 2에서 리스트의 하위태그로 li를 사용한다 -> list item의 줄임말
        
        cf. ol 태그는 속성에 type(순서의 모양), start(순서의 시작 번호)를 추가로 삽입할 수 있다.

    용어의 정의를 나열하는 태그
        어떤 리스트를 1:1로 매칭하는 경우가 있다고 가정했을 때, (apple-사과, banana-바나나, cloud-구름)
        용어와 정의는 1:1 대응이 가능하고, 혹은 n:n 여러 용어에 하나의 정의를 붙이거나,
        하나의 용어에 여러 개의 정의를 붙이거나, 마지막으로 여러애의 용어를 여러 개의 정의를 붙이는 것도 가능하다.

        1. <dl> : definition list의 줄임말. 정의 목록을 의미
        2. <dt> : definition term의 줄임말. 정의할 용어.
        3. <dd> : defition description의 줄임말. 용어의 설명. (dltag.html 참조)

1. 이미지 태그 사용하기 (n1imagetag.html 참조)
    <img src="(이미지 파일 경로)" alt="(대체 텍스트)" title="(툴팁 텍스트)">
    src : 필수 속성으로, source의 줄임말.
    alt : 이미지를 설명하는 문구로 로딩되지 않았을 경우에 나타나는 텍스트
    title : 이미지의 제목. 마우스를 갖다댔을 때 나오는 문구.

    src에서의 경로 표현
        절대 경로 : 이미 지정되어있는 명확한 경로를 명시한 것.
        상대 경로 : 특정 경로를 기준으로 상대적으로 변경될 수 있는 경로.
            상대 경로를 표시할 때의 ./ vs ../
            : ./ -> 현재 폴더, ../ -> 상위 폴더

3. 다른 곳으로 링크하기 (SpringBoot에서 보러가기 버튼)
    <a> 태그로 링크 이동하기
    형식 :
        <a href="(연결할 주소): target="(링크를 열 곳 옵션)">제목</a>

        속성 정리
            href : hypertext reference의 줄임말. 실제로 우리가 접속할 페이지의 주소
            target: 링크를 어떤 형태로 열 것인지에 관한 옵션
                _self : 보고 있는 창에서 넘어감
                _blank : 새 탭 / 새 창 -> 최근에는 이것이 많이 쓰인다.
